# Пошук у ширину (Breadth-First Search)

## 1. Основи графів: вершини, ребра, шляхи

**Граф** — це структура даних, яка складається з:
- **Вершин (вузлів)** — точок, що представляють об'єкти (наприклад, людей, міста, сторінки сайту).
- **Ребер (зв'язків)** — ліній, що з'єднують вершини та показують зв'язок між ними.

Приклад простого графа:

```
A —— B —— D
|    |
C —— F —— E
```

Тут вершини: `A, B, C, D, E, F`, а ребра: `A-B, A-C, B-D, B-F, C-F, F-E`.

### Основні поняття

| Термін | Опис |
|---|---|
| Орієнтований граф | Ребра мають напрямок (A → B ≠ B → A) |
| Неорієнтований граф | Ребра не мають напрямку (A-B = B-A) |
| Зважений граф | Кожне ребро має "вагу" (відстань, вартість) |
| Шлях | Послідовність вершин, з'єднаних ребрами |
| Цикл | Шлях, що повертається у початкову вершину |
| Степінь вершини | Кількість ребер, що виходять з вершини |
| Зв'язний граф | Граф, у якому з будь-якої вершини можна дістатись до будь-якої іншої |

## 2. Представлення графів у пам'яті

Існує кілька способів зберігати граф у програмі:

### 2.1. Матриця суміжності (Adjacency Matrix)

Двовимірна таблиця розміром `n × n`, де `n` — кількість вершин. Якщо є ребро між вершинами `i` та `j`, ставимо `1`, інакше `0`.

```
    A  B  C  D
A [ 0  1  1  0 ]
B [ 1  0  0  1 ]
C [ 1  0  0  0 ]
D [ 0  1  0  0 ]
```

- ✅ Швидка перевірка: чи існує ребро (`O(1)`).
- ❌ Витрачає багато пам'яті (`O(n²)`), навіть якщо ребер мало.

### 2.2. Список суміжності (Adjacency List)

Для кожної вершини зберігається список її сусідів.

```python
graph = {
    "A": ["B", "C"],
    "B": ["A", "D"],
    "C": ["A"],
    "D": ["B"]
}
```

- ✅ Економить пам'ять (`O(V + E)`), де `V` — вершини, `E` — ребра.
- ✅ Найпопулярніший спосіб для алгоритмів на графах, у тому числі BFS.
- ❌ Перевірка наявності ребра трохи повільніша, ніж у матриці.

### 2.3. Список ребер (Edge List)

Просто список усіх пар `(вершина1, вершина2)`:

```python
edges = [("A", "B"), ("A", "C"), ("B", "D")]
```

Зручний для зберігання, але неефективний для пошуку сусідів конкретної вершини.

## 3. BFS: алгоритм і його реалізація

### 3.1. Ідея алгоритму

**BFS (Breadth-First Search, пошук у ширину)** — це алгоритм обходу графа, який досліджує вершини **рівень за рівнем**: спочатку всі вершини на відстані 1 ребра від старту, потім на відстані 2, і так далі.

Головна структура даних — **черга (queue)**: вершини додаються в кінець і обробляються з початку (принцип FIFO — "перший прийшов, перший вийшов").

### 3.2. Покроковий алгоритм

1. Створити порожню чергу та додати в неї стартову вершину.
2. Позначити стартову вершину як "відвідану".
3. Поки черга не порожня:
   - Взяти вершину з початку черги.
   - Обробити її (наприклад, вивести).
   - Для кожного невідвіданого сусіда: позначити як відвіданого та додати в черту.
4. Повторювати, доки черга не стане порожньою.

### 3.3. Псевдокод

```
BFS(graph, start):
    visited = {start}
    queue = [start]

    while queue не порожня:
        vertex = queue.вилучити_з_початку()
        обробити(vertex)

        for neighbor у graph[vertex]:
            if neighbor not in visited:
                visited.додати(neighbor)
                queue.додати_в_кінець(neighbor)
```

### 3.4. Реалізація на Python

Повна реалізація з прикладами знаходиться у файлі **`main.py`**. Там реалізовано:
- `bfs(graph, start)` — обхід графа та порядок відвідування вершин;
- `print_levels(graph, start)` — виведення вершин по рівнях;
- `bfs_shortest_path(graph, start, goal)` — пошук найкоротшого шляху між двома вершинами.

### 3.5. Складність алгоритму

- **Часова складність:** `O(V + E)` — кожна вершина та кожне ребро обробляються один раз.
- **Просторова складність:** `O(V)` — для черги та множини відвіданих вершин.

## 4. Приклади використання BFS у житті

1. **Найкоротший шлях у незваженому графі** — наприклад, мінімальна кількість пересадок у транспортній мережі чи мінімальна кількість кліків між двома сторінками сайту.
2. **Соціальні мережі** — пошук "ступенів знайомства" (наприклад, функція "друзі друзів" або теорія "шести рукостискань").
3. **Веб-пошук та краулери** — пошукові системи обходять сторінки сайту рівень за рівнем, починаючи з головної сторінки.
4. **GPS-навігація та маршрутизація** — пошук маршруту з мінімальною кількістю перехресть/зупинок.
5. **Розв'язання головоломок та ігор** — наприклад, пошук мінімальної кількості ходів для розв'язання кубика Рубика, пазла "15" чи лабіринту.
6. **Мережеві протоколи** — поширення повідомлень у комп'ютерних мережах (broadcast), де пакет передається "хвилею" від вузла до сусідніх вузлів.

---

## Висновок

BFS — один із фундаментальних алгоритмів обходу графів, який гарантує знаходження найкоротшого шляху в незваженому графі завдяки своєму "пошаровому" принципу роботи через чергу. Розуміння представлення графів (список/матриця суміжності) є необхідною базою для реалізації BFS та інших алгоритмів на графах (DFS, Dijkstra, тощо).
